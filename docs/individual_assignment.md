| 2       | Формирование требований к системе          | 21 (с 02.03.2026 по 22.03.2026) | Определить требования к разрабатываемой системе автоматической генерации технической документации из исходного кода. Определить, какие функции должна выполнять система, какие задачи автоматизации она должна решать, а также какие ограничения накладываются на ее работу. Определить нефункциональные требования, связанные с особенностями работы системы: требования к производительности, масштабируемости, удобству использования, а также к формату входных и выходных данных. В результате составить текстовый документ, содержащий перечень функциональных требований к системе в виде структурированного списка, перечень нефункциональных требований, краткое описание сценариев использования системы, описание предполагаемых типов входных данных и форматов выходной документации. |

Описать структуру системы, определить состав ее модулей, а также их функции. Определить, каким образом будет осуществляться взаимодействие между компонентами системы и каким образом будет происходить передача данных между ними. Описать последовательность обработки данных внутри системы: от получения исходного кода до формирования итоговой документации. В результате составить документ, содержащий: описание архитектуры системы; структурную схему системы (архитектурную диаграмму); описание основных компонентов системы и их функций; описание потоков данных между компонентами системы; описание последовательности обработки данных. Разработать схему формирования входных данных для системы генерации текста, включая объединение информации о программных элементах и дополнительного контекста проекта. Результатом является документ, содержащий: описание метода извлечения информации из исходного кода; описание структуры данных для представления элементов программы; схему формирования входных данных для генерации документации; примеры структурированных входных данных, подготовленных на основе исходного кода.                                                                                                                                                      |

После реализации модулей необходимо выполнить их интеграцию в единый рабочий процесс, обеспечивающий последовательное выполнение всех этапов обработки данных. Создать исполняемый файл из разработанного исходного кода. Результатом является файл, содержащий описание структуры программной реализации системы; описание реализованных модулей и их функций; примеры входных данных и соответствующих им сгенерированных документов.                                                                                                                                                                                                                                                                                                                                                                                                                                     |
