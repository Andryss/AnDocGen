## Классы

### `class InMemoryStorage`

Простое хранилище в памяти для демонстрационных целей.

#### Методы

##### `__init__() -> None`

Инициализирует экземпляр хранилища в памяти, создавая пустой словарь данных и устанавливая начальный идентификатор.

##### `get(item_id: int) -> dict[str, Any] | None`

Метод возвращает элемент из внутреннего хранилища по его идентификатору. Если элемент с указанным идентификатором отсутствует, возвращается `None`.

**Параметры:**

- `item_id` (`int`) — идентификатор элемента, который требуется получить

**Возвращаемое значение:**

- `dict[str, Any] | None` — словарь с данными элемента, если он найден, иначе `None`

**Примеры:**

```python
storage = InMemoryStorage()
# предположим, что хранилище содержит элемент с id=1
result = storage.get(1)  # возвращает словарь с данными
result = storage.get(999)  # возвращает None
```

##### `create(payload: dict[str, Any]) -> int`

Создаёт новый элемент в хранилище, присваивая ему уникальный идентификатор, и возвращает этот идентификатор.

**Параметры:**

- `payload` (`dict[str, Any]`) — данные, которые будут сохранены в хранилище

**Возвращаемое значение:**

- `int` — уникальный идентификатор созданного элемента

**Побочные эффекты:**

Увеличивает внутренний счётчик `_next_id` и добавляет запись в словарь `_data`.

**Примеры:**

```python
storage = InMemoryStorage()
item_id = storage.create({"name": "example"})
```

## Функции

### `def normalize_email(email: str) -> str`

Нормализует адрес электронной почты для хранения: удаляет начальные и конечные пробелы и приводит к нижнему регистру.

**Параметры:**

- `email` (`str`) — адрес электронной почты для нормализации

**Возвращаемое значение:**

- `str` — нормализованный адрес электронной почты (без пробелов, в нижнем регистре)

**Примеры:**

Пример использования: `normalize_email("  User@Example.COM  ")` возвращает `"user@example.com"`.
