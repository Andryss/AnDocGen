# Модуль `storage.py`


Модуль предоставляет простое хранилище в памяти и утилиту для нормализации email-адресов.

- [Классы](#классы)
- [Функции](#функции)

## Классы

### `class InMemoryStorage`

Простое хранилище в памяти для демонстрационных целей.

#### Методы

##### `__init__() -> None`

Инициализирует хранилище в памяти, создавая пустой словарь для данных и устанавливая начальный идентификатор.

**Побочные эффекты:**

Инициализирует внутреннее состояние объекта: создает пустое хранилище `_data` и устанавливает `_next_id` в 1.

##### `get(item_id: int) -> dict[str, Any] | None`

Возвращает элемент из хранилища по указанному идентификатору, если он существует, иначе возвращает None.

**Параметры:**

- `item_id` (`int`) — идентификатор элемента, который необходимо получить из хранилища

**Возвращаемое значение:**

- `dict[str, Any] | None` — словарь с данными, если элемент найден, иначе None

##### `create(payload: dict[str, Any]) -> int`

Создаёт новый элемент в хранилище с уникальным идентификатором и сохраняет переданные данные.

**Параметры:**

- `payload` (`dict[str, Any]`) — данные, которые необходимо сохранить в хранилище

**Возвращаемое значение:**

- `int` — уникальный идентификатор созданного элемента

**Побочные эффекты:**

Изменяет состояние внутренних атрибутов экземпляра: увеличивает `_next_id` и добавляет новый элемент в словарь `_data`.

**Примеры:**

```python
storage = InMemoryStorage()
item_id = storage.create({"name": "test", "value": 42})
print(item_id)  # например, 0
```

## Функции

### `def normalize_email(email: str) -> str`

Нормализует адрес электронной почты для хранения, удаляя пробелы по краям и приводя к нижнему регистру.

**Параметры:**

- `email` (`str`) — адрес электронной почты, который необходимо нормализовать

**Возвращаемое значение:**

- `str` — нормализованный адрес электронной почты в нижнем регистре без пробелов по краям

**Примеры:**

```python
normalized = normalize_email("  Example@Domain.COM  ")
print(normalized)  # выведет: example@domain.com
```
