**Поля:**

- `status` (`int`) — —
- `body` (`dict[str, Any]`) — —

### `class UserHandler`

Обрабатывает операции API, связанные с пользователями.

#### Методы

##### `__init__(storage: InMemoryStorage) -> None`

Инициализирует экземпляр UserHandler с указанным хранилищем.

**Параметры:**

- `storage` (`InMemoryStorage`) — хранилище для управления данными пользователей

**Побочные эффекты:**

Присваивает переданное хранилище приватному атрибуту экземпляра `_storage`.

**Примеры:**

```python
storage = InMemoryStorage()
handler = UserHandler(storage)
```

##### `get_user(user_id: int) -> ApiResponse`

Извлекает пользователя по его идентификатору из хранилища. Если пользователь не найден, возвращает ответ с кодом 404, иначе — с кодом 200 и данными пользователя.

**Параметры:**

- `user_id` (`int`) — уникальный идентификатор пользователя, который необходимо получить

**Возвращаемое значение:**

- `ApiResponse` — объект ответа, содержащий статус и тело с данными пользователя или сообщением об ошибке

**Примеры:**

```python
handler = UserHandler(storage=InMemoryStorage())
response = handler.get_user(123)
if response.status == 200:
    print(response.body)
else:
    print("Пользователь не найден")
```

##### `create_user(name: str, email: str) -> ApiResponse`

Создаёт нового пользователя с заданными именем и email, сохраняет его в хранилище и возвращает ответ с идентификатором пользователя.

**Параметры:**

- `name` (`str`) — имя пользователя
- `email` (`str`) — адрес электронной почты пользователя

**Возвращаемое значение:**

- `ApiResponse` — объект ответа с кодом состояния 201 и телом, содержащим идентификатор созданного пользователя

**Побочные эффекты:**

Сохраняет нового пользователя в хранилище `self._storage` и нормализует email перед сохранением.

**Смотрите также:**

- `normalize_email` — нормализует адрес электронной почты для хранения, удаляя пробелы по краям и приводя к нижнему регистру.
